The crash is caused by **one specific misunderstanding of `customtkinter` themes**.

## ‚ùå The Problem (why it crashes)

This line:

```python
set_default_color_theme("#FF0000")
```

`set_default_color_theme()` **does NOT accept hex colors**.

Internally, CustomTkinter treats the argument as **a filename**, so it literally tries to open:

```
"#FF0000"
```

‚Üí which obviously does not exist
‚Üí hence:

```
FileNotFoundError: No such file or directory: '#FF0000'
```

---

## ‚úÖ What `set_default_color_theme()` actually accepts

Only **theme names** or **JSON theme files**:

### Built-in themes

```python
set_default_color_theme("blue")
set_default_color_theme("green")
set_default_color_theme("dark-blue")
```

### OR a JSON file

```python
set_default_color_theme("my_theme.json")
```

---

## ‚úÖ Correct Fix (pick ONE)

### ‚úî Option 1 ‚Äî Use a built-in theme (simplest)

```python
set_appearance_mode("dark")
set_default_color_theme("dark-blue")
```
